# Presets and State Serialization

Factory presets, state saving/loading, and preset management.

## Factory Presets

Create baked-in presets in the plugin constructor, after parameter initialization.

### MakePreset (Positional)

Values are listed in parameter index order. All parameters must be specified:

```cpp
// Params: kGain, kFreq, kQ
MakePreset("Flat",     0.,   1000., 0.707);
MakePreset("Bass",     6.,   200.,  1.0);
MakePreset("Treble",  -3.,   8000., 0.5);
```

### MakePresetFromNamedParams

Specify only the parameters you want to set. Unspecified parameters get their default values:

```cpp
MakePresetFromNamedParams("Deep Bass", 2,
  kGain, 12.,
  kFreq, 80.
);  // kQ uses its default value
```

### MakeDefaultPreset

Fill remaining preset slots with default values:

```cpp
// Create 3 named presets, then fill 5 more with defaults
MakePreset("Clean", 0., 1000., 0.707);
MakePreset("Warm", 3., 500., 1.2);
MakePreset("Bright", -2., 5000., 0.5);
MakeDefaultPreset("Init", 5);
```

### MakePresetFromChunk

For plugins with complex state (beyond just parameters):

```cpp
IByteChunk chunk;
SerializeState(chunk);  // capture current state
MakePresetFromChunk("Complex Preset", chunk);
```

### MakePresetFromBlob

For embedding presets as base64 strings (generated by `DumpPresetBlob`):

```cpp
MakePresetFromBlob("Factory 1", "AAAA...", sizeOfChunk);
```

## Preset Debugging Utilities

These methods write preset code to a file, useful for creating factory presets from the current plugin state:

```cpp
// Write MakePreset() call for current state
DumpMakePresetSrc("/path/to/output.txt");

// Write MakePresetFromNamedParams() call
const char* paramNames[] = {"kGain", "kFreq", "kQ"};
DumpMakePresetFromNamedParamsSrc("/path/to/output.txt", paramNames);

// Write MakePresetFromBlob() call (for complex state)
DumpPresetBlob("/path/to/output.txt");
```

These are typically called from a debug button or menu action during development.

## State Serialization

### Basic (Parameters Only)

The default implementation serializes all parameter values:

```cpp
// Serialize all parameter values to a chunk
bool SerializeParams(IByteChunk& chunk) const;

// Unserialize parameter values from a chunk
int UnserializeParams(const IByteChunk& chunk, int startPos);
```

`SerializeState` and `UnserializeState` call these by default.

### Custom State

Override `SerializeState` / `UnserializeState` for plugins with extra state beyond parameters (e.g., wavetables, sample data, user settings):

```cpp
bool SerializeState(IByteChunk& chunk) const override
{
  // Serialize custom data first
  chunk.Put(&mCustomData);
  chunk.PutStr(mUserLabel.Get());

  // Then serialize parameters
  return SerializeParams(chunk);
}

int UnserializeState(const IByteChunk& chunk, int startPos) override
{
  // Unserialize custom data first
  startPos = chunk.Get(&mCustomData, startPos);
  startPos = chunk.GetStr(mUserLabel, startPos);

  // Then unserialize parameters
  return UnserializeParams(chunk, startPos);
}
```

**Important:** Always call `SerializeParams`/`UnserializeParams` to handle parameter state. Custom data should be serialized before parameters.

### IByteChunk API

```cpp
class IByteChunk {
  // Writing
  int Put(const T* pVal);                    // Put any POD type
  int PutStr(const char* str);               // Put null-terminated string
  int PutChunk(const IByteChunk* pRHS);      // Append another chunk
  int PutBytes(const void* pBuf, int size);  // Put raw bytes

  // Reading
  int Get(T* pVal, int startPos);            // Get any POD type
  int GetStr(WDL_String& str, int startPos); // Get string
  int GetBytes(void* pBuf, int size, int startPos);

  // Utilities
  int Size();
  void Clear();
  uint8_t* GetData();
};
```

### State Chunks vs Default Behavior

If `PLUG_DOES_STATE_CHUNKS` is `true` in `config.h`, the plugin handles its own state serialization via `SerializeState`/`UnserializeState`. If `false`, the host manages parameter state directly (simpler but less flexible).
